How to Use

1. Add devices using the form above — each device represents a WWVDS camera unit in the field.

2. Select a device in the sidebar, then go to Generate Events to simulate alerts and image updates.

3. The Event Log tab shows all alerts and updates received by the simulated SunGuide server in real-time.

4. Use Device Status to query the GET /v1/status endpoint for any configured device.

5. Configure an external SunGuide URL in Settings to forward generated messages to a real SunGuide instance.

WWVDS SunGuide HTTP Protocol — Quick ReferenceRev 3.0 · March 2026

1. Device Status

SunGuide polls the WWVDS device to get its current operational status.

GET http[s]://<device-ip>/v1/status?DeviceId={deviceId}

FieldTypeRequiredDescription
deviceIdstringYESUnique ID of the device
deviceStatusstringYESActive – operational & detecting
Error – detection non-functional
Out of Service – intentionally non-functional
deviceTimestampdateTimeYESISO 8601 timestamp (e.g. 2021-06-15T13:45:30.0000000-07:00)

2. Wrong-Way Alert

Sent immediately when a wrong-way vehicle is detected. Device POSTs to SunGuide.

POST http[s]://<sunguide-ip>/v1/alert

FieldTypeRequiredDescription
alertIdstringYESUnique ID for this alert
deviceIdstringYESDevice that generated the alert
alertTimestampdateTimeYESISO 8601 timestamp
imageListlistoptional0–10 image URLs available at alert time
imageLocationstringoptionalURL to download an image
roadwaystringoptionalRoadway name (requires direction)
directionstringoptionalNorthbound · Eastbound · Southbound · Westbound · Innerloop · Outerloop
(requires roadway)

3. Image Update

Sent multiple times as additional images become available after the alert. Device POSTs to SunGuide.

POST http[s]://<sunguide-ip>/v1/update

FieldTypeRequiredDescription
alertIdstringYESAlert ID this update belongs to
deviceIdstringYESDevice that generated the original alert
updateTimestampdateTimeYESISO 8601 timestamp
imageListlistYES1–10 image URLs (mandatory in update)
imageLocationstringYESURL to download an image

HTTP Response Codes

CodeMeaning
200 OKSuccessful response
400 Bad RequestMalformed request or missing required fields
404 Not FoundDevice not found (status endpoint)
500 Internal Server ErrorServer-side error
